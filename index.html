Demonstrating AI-powered visual regression testing using Applitools Eyes with Selenium 4 and TestNG. Establishes a UI baseline, then catches a CSS regression — a button colour change simulating a bad deploy — flagged automatically by the Visual AI engine.

Demo

Visual regression — login page
Repo

Two TestNG tests target the Applitools demo app. The baseline test captures the correct UI and establishes the approved reference. The regression test injects a CSS change — turning the login button red — and Applitools Visual AI flags the diff against the approved baseline, marking the test as Unresolved.

Java Selenium 4 TestNG Applitools Eyes Visual AI WebDriverManager Maven

Flow

Establish baseline→ Approve in dashboard→ Inject CSS regression→ Run test→ AI flags diff

How to run

01

Run the baseline test

Selenium opens the demo login page and Applitools captures a screenshot. The snapshot is sent to your Applitools account and appears in the dashboard with the status New — no approved reference exists yet.

mvn test -Dtest=loginPageBaseline
02

Approve the baseline in the Applitools dashboard

Log in at eyes.applitools.com, open the test run, and click Accept on the snapshot marked New. This tells Applitools "this is the correct UI — use it as the reference for all future comparisons." Without this step the regression test has nothing to compare against.

03

Run the regression test

A CSS change is injected at runtime, turning the login button red to simulate a bad deploy. Applitools compares the new screenshot against the approved baseline and marks the test Unresolved, with the diff highlighted in the dashboard.

mvn test -Dtest=loginPageRegression

Running mvn test without specifying a test executes both — skip the approval step and both snapshots show as New rather than flagging the diff.
